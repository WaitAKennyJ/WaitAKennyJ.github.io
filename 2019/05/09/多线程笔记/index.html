多线程笔记

Posted on 2019-05-09 | In Java , J2SE | Comments:

基础

  1. synchronized锁定的是一个对象，而不是一段代码。

    和

    是等同的。

  2. synchronized锁定的是一个静态对象的时候，是锁定的对象的class。静态对象不需要new出来，所以没有this引用。

    和

    是等同的。

  3. synchronized锁是可重入的。

    一个同步方法可以调用另一个同步方法。​​​​​​
    一个线程已经拥有某个对象的锁，再次申请时仍会得到该对象的锁。（同一线程，同一把锁）

  4. 子类重写父类synchronized锁方法时，可以调用父类的锁方法。

  5. 程序在执行过程中，如果出现异常，默认情况下锁会被释放。所以应对会出现异常的代码块try-catch处理

  6. volatile使变量在线程之间可见（修改时会提醒线程变量修改，让线程将变量值读入CPU缓冲区），但volatile也仅能保证可见性，无法代替synchornized

    编译器为了加快程序运行速度，对一些变量的写操作会现在寄存器或CPU缓存上进行，最后写入内存。而在这个过程中，变量的新值对其它线程是不可见的。

  7. Atom类的方法之间并不具有原子性

  8. Sychronized锁定的对象被改变，锁将被释放

    不要用字符串常量作为锁定对象

  9. Reentrantlock用于替代Sychronized，更灵活，但是需要手动释放锁。

    并且可以尝试trylock，这样无法锁定，或者短时间内无法锁定，线程可以决定是否继续等待。
    还可以调用lockInterruptibly()方法（lock无法响应），可以响应线程的interrupt方法。
    还可以指定为公平锁new Reentrantlock(true) true 表示公平锁，等待时间越长，就能得到锁
    lock.newCondition()可以精确await、signal

  10. ThreadLocal是线程局部变量

并发容器

map/set：

  • Hashmap
  • Treemap
  • Linkedhashmap
  • Hashtable
  • Collections.sychronizedXXX
  • Concurrenthashmap
  • Concurrentskiplistmap（跳表，是排好序的，插入较慢）

list：

  • ArrayList
  • LinkedList
  • Vector
  • Collections.sychronizedXXX
  • CopyOnWriteList （读的时候较多时，eg.监听器队列）

Queue:

  • ConcurrentLinkedQueue

  • BlockingQueue（生产者消费者中的容器）

    • ArrayBlockingQueue（有界队列）
    • LinkedBlockingQueue（无界队列）
    • TransferQueue（eg.游戏服务器接受消息）
    • SynchronusQueue（容量为0的TransferQueue）
    • DelayQueue（执行定时任务）

线程池

  • Executor （一个顶层接口） 实现excute(Runnable command)方法
  • ExecutorService 实现submit(Runnable/Callable command)方法
  • Executors （一个操作Executor等类的操作类）
  • Future（有返回值）

  • ThreadPool

    • newFixedThreadPool (固定的)
    • newCachedThreadPool（根据任务弹性生成线程，超过一定时间未使用则关闭）
    • newSingleThreadExecutor（保证任务执行顺序）
    • newScheduledPool（计划线程池）
    • newWorkStealingPool ->守护线程（每个线程有自己的任务队列，完成自己的后会偷取别的线程的任务）
    • ForkJoinPool ->守护线程 类似MapReduce
