关于算法中的逆元问题

Posted on 2019-04-29 | In Algorithm

当题目要求对结果求m的模时,如果式中没有出现除法,则可以使用同余模公式。

同余模公式

而当过程中需要计算a/b时,需要对a/b取模,即(a/b) mod m，有时会出现爆精度的情况,所以需要变除法为乘法,即逆元。

逆元的定义

方程ax≡1(mod p),的解称为a关于模p的逆，当gcd(a,p)==1（即a，p互质）时，方程有唯一解，否则无解。

对于一些题目会要求把结果MOD一个数，通常是一个较大的质数，对于加减乘法通过同余定理可以直接拆开计算，

但对于(a/b)%MOD这个式子，是不可以写成(a%MOD/b%MOD)%MOD的，但是可以写为(a*b^-1)%MOD,其中b^-1表示b的逆元。

逆元的解法

  • 费马小定理。
    假如p是质数，且gcd(a,p)=1，那么$ a^{(p-1)}≡1（mod\; p）$，即：假如a是整数，p是质数，且a,p互质(即两者只有一个公约数1)，那么a的(p-1)次方除以p的余数恒等于1。
    由上方的公式很容易推导出:$a*a^{(p-2)}≡1（mod\;
    p）$对于整数a，p，a关于p的逆元就是$a^{(p-2)}$,直接快速幂解之即可。

  • 扩展欧几里德算法

  • 线性求逆元

说明一下费马小定理中的幂解

我们知道对于像 7%2,3%5这样的题,计算机很容易算出它们的结果,但是如果我们需要计算 $7^{123456789}$ %65536这样的值呢,这时普通的计算方式可能就要花费很久的时间了,有没有简单的方法可以算出来这类大数的模呢?

  $a=im+jb=sm+t$
所以有
  $a⋅b=(im+j)⋅(sm+t)=$ism^2$+jsm+itm+jt$

对于像$7^{123456789}%65536$这样的大数幂模运算,我们可以把大数幂m拆分成形如$m=m_1+m_2+…+m_n$这样的形式,那么$n^m=n^{m_1+m_2+…+m_n}=n^{m_1}n^{m_2}⋅…n^{m_n}$,这样通过上面的公式我们就可以很容易算出大数幂了.

前言

我的毕业设计是，基于hadoop的电影推荐系统。
我的导师反正就很不负责的甩给我一堆不知道哪里来的PPT和两个虚拟机镜像让我自己搞去。
因为我用的是MBP，没有买U盘的转接口，这个镜像也用不了，那就只能自己搞了呗。
我Mac选择的虚拟机是Parallel Desktop，装的系统是CentOS7.

  • 坑点1
    刚开始是用Mac做master，CentOS做slave。
    master上装hadoop是用的homebrew。

    然后他就给我装了3.1.1的版本，网上的配置都很乱，配置了半天才发现homebree给我装的hadoop的文件夹根目录下面，根本就没有 /etc/hadoop/ 这个文件夹，我配置了半天都配置错了。
    我配置的都是/libexec/etc/hadoop/下的四个site.xml文件。
    最后从官网上下了个压缩文件才发现的。

  • 坑点2
    一定要在core-site.xml里配置hadoop.tmp.dir属性，这个属性是hadoop存储一些临时文件的目录，name和data文件夹的默认存储目录也是根据这个tmp目录来的。
    如果不指定它的位置，hadoop对它设置的默认位置，将会在/var下(应该是这里 XD)，每次关机系统都会清理这里的数据，所以后面会出现一些莫名的错误。

  • 坑点3
    不要给你的配置文件添加一些奇怪的属性，最基础的就好。
    比如我配置好之后运行mapreduce自带的example，运算PI值得时候，他就会有一个奇怪的现象。

    然后计算不出结果。
    百度说要修改虚拟机的内存，改成至少1G，但是我本就是分配了2G呀，总不能多了还不好吧。
    后来发现是我根据百度的教程在core-site.xml里配置了一个属性。

    把它在master和slave里都删了，PI值运算成功。

  • 坑点4
    我在mac上配置hadoop遇到最大的坑点就是上传文件不成功。
    而且我现在也不算完全解决。

    之前我是在Mac上做master，虚拟机做slave，结果master不能上传文件，但是slave可以，且两者都不能运行example。

    终端错误如下：

    查了很多资料说是防火墙问题或者ssh问题。
    但是我Mac防火墙本身就是关闭的，虚拟机也被我关闭了，最后还是不行，我都要绝望了，格式化了n遍hdfs，都不能成功。
    最后我克隆了slave，当master，一切迎刃而解。
    可能问题确实是出来网络方面，Mac哪里没有设置对，我现在还不得而知。

Java String 总结

Posted on 2019-04-07 | Edited on 2019-04-08 | In Java , J2SE

String简介

String定义:

不可变String

String对象是不可变的。查看JDK文档你会发现，String类中每一个看起来会修改String值的方法，实际上都是创建了一个全新的的String对象，以包含修改后的字符串内容。而最初的String对象则丝毫未动。
String设计为不可变类主要考虑到：效率和安全。

  • 效率：
    1.在早期的JVM实现版本中，被final修饰的方法会被转为内嵌调用以提升执行效率。而从Java SE5/6开始，就渐渐摈弃这种方式了。因此在现在的Java SE版本中，不需要考虑用final去提升方法调用效率。只有在确定不想让该方法被覆盖时，才将方法设置为final。
    2.缓存hashcode，String不可变，所以hashcode不变，这样缓存才有意义，不必重新计算。
  • 安全：
    String常被作为网络连接，文件操作等参数类型，倘若可改变，会出现意想不到的结果。

String的创建方式

从上面的题中你会知道，String的创建方式有两种：

  • 直接赋值
    • 此方式在方法区中字符串常量池中创建对象

分析

让我们先来了解一下JVM内存中的栈(stack)、堆(heap)和方法区，图如下。

str1 == str2

当执行第一句时，JVM会先去常量池中查找是否存在HelloKennyJ，当存在时直接返回常量池里的引用；当不存在时，会在字符创常量池中创建一个对象并返回引用。
当执行第二句时，同样的道理，由于第一句已经在常量池中创建了，所以直接返回上句创建的对象的引用。

str1 == str3

执行第一句，同上第一句。

执行第二句时，会在堆（heap）中创建一个对象，当字符创常量池中没有”HelloKennyJ”时，会在常量池中也创建一个对象；当常量池中已经存在了，就不会创建新的了。

str1 == str6

由于”Hello”和”KennyJ”都是常量，编译时，第二句会被自动编译为String str6 = “HelloKennyJ”;

str1 == str7

其中前三句变量存储的是常量池中的引用地址。

第四句执行时，JVM会在堆（heap）中创建一个以str4为基础的一个StringBuilder对象，然后调用StringBuilder的append()方法完成与str5的合并，之后会调用toString()方法在堆（heap）中创建一个String对象，并把这个String对象的引用赋给str7。
也就是说str7和str1是两个不同的对象。

String相关

由于String的不可变性导致，字符串变更时效率低下，在之后得JDK版本中出现了StringBuilder和StringBuffer 。

类 可变性 线程安全
String 不可变 安全
StringBuilder 可变 非安全
stringBuffer 可变 安全

使用选择

1、当有少量连接操作时，使用String
2、当单线程下有大量连接操作时，使用StringBuilder
3、当多线程下有大量连接操作时，使用StringBuffer
